# 关于本文件

记录一些个人的思考和 CS 学习路上的经验

# 用 markdown 记笔记

- markdown 可以以高效的,以自带类似思维导图的层次去记录学习的知识
- 记录学到的知识可以给自己的学习提供专注力,以及正向反馈,提高学习动力
- 笔记是给未来的自己翻阅所拥有技术栈的最佳文档,可以说是自己的`技能书`

# CS 学习心得

- make hands dirty,走出舒适圈的实践学习 
- 觉得知识难学的原因往往是缺少一份通俗,深入浅出的学习资料
- 与已经登堂入室的高手交流，能减少踩坑，更快登堂入室
- 保持专注，需要习惯

# 代码心得

- 要把复杂庞大的任务分解成小任务去完成
- 编程最伟大的地方在于封装底层
- 写注释去完成整体架构，再写代码

# 人生思考

- 要去提升自己能为别人提供的价值 
- 人有惰性，尽可能简单又有效
- 合适的生活节奏，白天一半时间学习一半时间工作，晚上一半时间学习，一半时间娱乐
